# Survival OP Server Configuration

Este repositorio contiene un sistema completo de configuraciones para servidores de Minecraft basados en la modalidad **Survival OP**. El objetivo de este proyecto es demostrar cómo diseñar una experiencia de juego completa mediante configuraciones avanzadas de plugins, estructuras de progresión y sistemas de economía balanceados.

## Descripción general

Survival OP es una variante avanzada del modo supervivencia tradicional donde los jugadores tienen acceso a herramientas, encantamientos y recompensas significativamente más poderosas que en el juego base. Sin embargo, para mantener el equilibrio del servidor, el sistema incluye múltiples capas de progresión y limitaciones estratégicas.

Este proyecto está diseñado para demostrar:

- estructura profesional de configuraciones
- integración entre plugins
- balance de progresión
- diseño de gameplay

## Sistema de progresión

La progresión del servidor está basada en un sistema de rangos que permite a los jugadores desbloquear nuevas habilidades, kits y beneficios a medida que avanzan.

Progresión de ejemplo:

Player

Iron  
Gold  
Emerald  
Diamond  
Netherite  
Titan  
Overlord  
Immortal  
Celestial  
Eternal  
Lunar  
Legendary  
Sultan  
Salik  
God

Cada rango ofrece beneficios únicos como:

- acceso a nuevos kits
- aumento de límites de homes
- acceso a comandos especiales
- mejoras económicas
- ventajas en PvP

## Sistema de economía

La economía del servidor está diseñada para evitar inflación y mantener el valor de los recursos.

Las principales fuentes de ingresos incluyen:

- venta de minerales
- recompensas por eventos
- comercio entre jugadores
- recompensas por PvP
- misiones diarias

También se incluyen mecanismos de control económico como:

- límites de venta
- impuestos del mercado
- eventos económicos especiales

## Sistema PvP

El combate jugador contra jugador es una parte importante del servidor. Para mantener el equilibrio competitivo se han implementado:

- arenas PvP dedicadas
- recompensas por rachas de kills
- eventos de combate
- zonas PvP específicas

El objetivo es incentivar la competencia sin afectar negativamente la experiencia de supervivencia.

## Encantamientos personalizados

El servidor incluye encantamientos personalizados diseñados para expandir las posibilidades de combate y progresión.

Ejemplos de encantamientos:

- Lifesteal
- Berserk
- Lightning
- Ice Aspect
- Demon Forged

Estos encantamientos permiten crear combinaciones únicas de equipamiento y estilos de combate.

## Scoreboards dinámicos

El sistema incluye scoreboards dinámicos que muestran información relevante del jugador.

Ejemplo de datos mostrados:

- nombre del jugador
- rango actual
- dinero
- nivel
- estadísticas PvP
- jugadores online
- eventos activos

## Eventos del servidor

Para mantener la actividad del servidor se incluyen eventos automáticos como:

- meteoritos
- invasiones de mobs
- jefes especiales
- torneos PvP

Estos eventos generan recompensas únicas que impulsan la economía y la progresión del jugador.

## Plugins utilizados

Este sistema está diseñado para funcionar con plugins ampliamente utilizados en servidores profesionales:

- LuckPerms
- PlaceholderAPI
- Vault
- TAB
- AdvancedEnchantments

## Objetivo del proyecto

Este repositorio busca demostrar conocimientos avanzados en configuración de servidores de Minecraft, incluyendo organización de archivos YAML, integración de sistemas de juego y diseño de experiencias multijugador equilibradas.
